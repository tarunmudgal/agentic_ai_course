# QA to AI Engineering Notes (Seed Corpus)

QA engineers moving to AI should reuse core strengths:

- Test design: convert requirements into prompt tests and evaluation assertions.
- Defect triage: classify hallucinations, retrieval misses, and formatting failures.
- Regression discipline: track baseline prompts and compare model behavior across versions.
- Risk-based strategy: prioritize high-impact workflows for guardrails and monitoring.

AI-specific additions:

- Grounding checks for RAG-based answers.
- Prompt and context versioning.
- Cost and latency budget checks in CI-style evaluation runs.
